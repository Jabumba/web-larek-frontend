Описание проекта
Проект "web-larek" представляет собой реализацию интернет-магазина для фронтенд разработчиков. Пользователь может просматривать товары, указывать способ оплаты, адрес, почту, телефон и делать заказ. Проект реализован на TypeScript с использованием API для получения товаров и отправки запроса на заказ. До оформления заказа содержимое корзины сохраняется в localStorage



Описание интерфейса
Интерфейс можно условно разделить на 3 процесса:
1) Просмотр товаров (MainScreen)
2) Добавление товара (ProductScreen, BasketScreen)
3) Оформление заказа (AddPayScreen, OrderScreen SuccessScreen)



Структура проекта в './types'
1) './global.d.ts' - Расширение глобального интерфейса NodeJS.ProcessEnv
2) './html.ts' - настройка html элементов
3) './settings.ts' - глобальные настройки всего проекта
4) './components./base' - базовые элементы отображения и событий
5) './components./model' - модель проекта
6) './components./view./common' - базовые компоненты отображения
7) './components./view./partial' - отображения данных
8) './components./view./screen' - верхнеуровненые отображения (экраны) в проекте



Архитектура проекта (MVC)
Описана единая модель данных приложения в файле src/types/components/model/app-state.ts, содержащая всю логику работы с данными и возможные действия над ними. Все изменения данных происходят через методы модели, а она в свою очередь уведомляет об изменениях через метод настроек onChange(changes: AppStateChanges).

При обработке событий возникающих в AppStateEmitter производится обновление данных в верхнеуровневых отображениях (экраны). Экраны это фактически крупные сборки инкапсулирующие детали реализации интерфейса и принимающие из вне только обработчики событий и необходимые данные. Экраны внутри составлены из более мелких отображений, которые инициализируют с помощью глобальных настроек проекта и распределяют данные между вложенными отображениями через свойства и метод render().



Базовый код
Класс EventEmitter позволяет подписываться на события и уведомлять подписчиков
о наступлении события.
1) on - подписка на событие
2) off - отписка от события
3) emit - уведомление о событии
4) onAll - подписка на все события
5) offAll - удаление всех подписчиков
6) trigger - генерирует событие с заданными агрументами. Это позволяет передавать его в качестве обработчика события в другие классы. Эти классы будут генерировать события, не будучи при этом напрямую зависимыми от класса  EventEmitter.



Модели
Вся глобальная модель данных приложения представлена классом AppState, которая содержит в себе все данные и методы, необходимые для работы с приложением. Модель уведомляет об изменениях через метод onChange(changes: AppStateChanges). В конструкторе она принимает первым параметром api для работы с сервером, а вторым параметром настройки (реализуют интерфейс IAppStateSettings).
AppState содержит следущие поля:
1) products - список товаров, представленных на глвной странице
2) selectedProducts - товары, добавленные в корзину
3) basketTotal - число товаров в корзине
4) basketPrice - цена всех товаров в корзине
5) userSettings - пользовательские настройки (данные) при оформлении заказа
6) completeOrder - ответ от сервера при совершении заказа
7) order - заказ, отправляемый на сервер для оформления
8) openedModal - открытое в данный момент модальное окно
9) isOrderReady - состояние заказа (можно ли его совершить)
10) modalMessage - текст ошибки
11) isError - состояние ошибки
И методы:
1) getProducts - получение товаров через api
2) orderProducts - соверешние заказа и отправка данных на сервер через api
4) selectProduct - добавление товара в корзину
5) removeProduct - удаление товара из корзины
6) fillSettings - обновление пользовательских настроек
7) isValidSettings - метод проверки пользовательских настроек на валидность
8) openModal - открытие модального окна
9) setMessage -

IAppStateSettings содержит настройки модели данных приложения, которые передаются в конструктор модели данных. Для удобства работы с данными в модели реализованы методы для изменения данных, которые в свою очередь вызывают метод onChange() через переданнеы настройки

IAppStateConstructor - конструктор модели данных.

Тип IModalChange содержит состояние приложения. Для корректной обработки событий открытия и закрытия модальных окон нам нужно знать предыдущее и текущее состояние.

список AppStateModals содержит возможные для открытия модальные окна.

список AppStateChanges содержит в себе события, происходящие в приложении.



Отображения в проекте разделены на три типа:
common — общие компоненты, не зависящие от доменной области проекта
partial — частичные компоненты, реализующие доменную область проекта
screen — верхнеуровневые компоненты, которые являются экранами приложения

Первые два типа (common и partial) независимо типизированы, не используют глобальных настроек напрямую и могут быть легко переносимы между проектами. Экраны (screen) же зависят от глобальных настроек и используют их для инициализации и передачи данных между вложенными отображениями, так как по факту это соединительный код для удобства вынесенные в отдельные файлы и оформленный как отображение.

IView - это базовый компонент всех отображений, он содержит поле element для хранения корневого элемента и несколько методов для настройки html элемента:
1) copy для копирования экземпляра класса
2) render для вывода готового html элемента
3) setHidden скрывает элемент, принимая его на вход
4) setText устанавливает текстовое содержимое, принимая на вход элемент и значение
5) setDisabled изменяет статус блокировки, принимая на вход элемент и булевое значение
6) toggleClass переключает класс. На вход элемент, класс, булевое значение
7) setText установить текстовое содержимое. На вход: элемент, значение текста
8) init устанавливает события на элемент
IModalScreenSettings - это компонент, содержащий базовые методы для каждого экрана в проекте для закрытия и следующего шага



Ключевые типы данных
IProduct содержит информацию о товаре в виде объекта, приходящей с сервера, которую мы используем в корзине и на главной странице.
IUser содержит информацию о данных пользователя и данных заказа, которые мы используем в модальных окнах при оформлении заказа.
OrderResponse - объект ответа сервера, представляющий заказ.
